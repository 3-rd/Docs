# AI 编程 Agent 深度对比：Claude Code vs OpenCode vs OpenClaw

> 整理时间：2026-03-30

---

## 一、定位与愿景

### Claude Code
**定位**：Anthropic 官方出品的编程 Agent，专注"让 AI 帮人写代码"。

**愿景**：成为一个无声的结对编程伙伴——快速响应、紧凑反馈、小步迭代。

### OpenCode
**定位**：开源社区驱动的编程 Agent，强调彻底性和开源透明。

**愿景**：让每个人都能免费使用强大的 AI 编程工具，模型无关、开源透明。

### OpenClaw
**定位**：个人 AI 助手框架，编程是核心技能之一，但不是全部。

**愿景**：成为个人数字化的"第二大脑"，跨时间、跨任务记住一切，主动服务而非被动响应。

## 二、技术架构

### Claude Code

```
┌─────────────────────────────────────────────────────┐
│                    Claude Code CLI                    │
│                                                      │
│  ┌──────────┐   ┌──────────┐   ┌──────────────┐  │
│  │  Claude  │◄──│  Tool    │◄──│   MCP Host   │  │
│  │   API    │──►│ Executor │──►│  (your infra)│  │
│  └──────────┘   └──────────┘   └──────────────┘  │
│                      │                              │
│              ┌───────┴───────┐                    │
│              │ Built-in Tools │                    │
│              │ - Read/Write   │                    │
│              │ - Bash/grep   │                    │
│              │ - Git         │                    │
│              │ - Search      │                    │
│              └───────────────┘                    │
└─────────────────────────────────────────────────────┘
```

**核心特点**：
- **单进程 CLI**：一个命令行工具，轻量简洁
- **原生工具集**：内置读文件、执行命令、Git 操作、代码搜索，工具集窄而精
- **MCP 扩展**：支持 MCP（Model Context Protocol）连接外部工具
- **模型专用**：调用 Claude API，与 Anthropic 深度绑定
- **会话内存**：单次会话内的上下文保持，关闭后丢失

**源码可见性**：闭源，Claude Code 本体不开源，但 Anthropic 公开了 Tool Use 的设计理念

```
┌─────────────────────────────────────────────────────────────┐
│                    OpenCode Architecture                     │
│                                                              │
│   ┌─────────────┐                    ┌─────────────┐       │
│   │   OpenCode  │◄─── gRPC/REST ───►│   LLM API   │       │
│   │   Client    │                    │ (任意模型)   │       │
│   │  (TUI/Web) │                    └─────────────┘       │
│   └──────┬──────┘                                             │
│          │                                                      │
│   ┌──────▼──────┐                    ┌─────────────┐       │
│   │   OpenCode  │◄─── WebSocket ───►│   MCP       │       │
│   │   Server    │                    │   Servers   │       │
│   │   (Go)      │                    └─────────────┘       │
│   └──────┬──────┘                                             │
│          │                                                      │
│   ┌──────▼──────────────────────────┐                        │
│   │        Tool Executor             │                        │
│   │  Read / Write / Bash / Git / ... │                        │
│   └─────────────────────────────────┘                        │
└─────────────────────────────────────────────────────────────┘
```

**核心特点**：
- **C/S 架构**：Go 写的服务端，长期驻留后台；客户端可以是 TUI、桌面 App 或 IDE 插件
- **模型无关**：内置免费模型，支持 75+ LLM 提供商（OpenAI / Claude / Gemini / MiniMax 等）
- **后台运行**：可以整晚通宵跑任务，AI 写代码同时你在睡觉
- **MCP 原生**：内置 MCP 支持，工具扩展能力极强
- **Slash Commands**：类似 IDE 的快捷命令系统，自动化工作流
- **开源**：完全开源，架构透明

### OpenClaw

```
┌──────────────────────────────────────────────────────────────┐
│                     OpenClaw Gateway                          │
│                                                               │
│   ┌────────────────────────────────────────────────────┐    │
│   │                   Agent Engine                       │    │
│   │   ┌──────────┐  ┌──────────┐  ┌──────────────┐  │    │
│   │   │ Memory   │  │  Skills   │  │   Tools      │  │    │
│   │   │  System  │  │ (Skill.md)│  │  (Function   │  │    │
│   │   │          │  │           │  │   Calling)   │  │    │
│   │   └──────────┘  └──────────┘  └──────────────┘  │    │
│   └────────────────────────────────────────────────────┘    │
│                         │                                     │
│   ┌─────────────────────▼───────────────────────────────┐    │
│   │              Multi-Channel Delivery                  │    │
│   │   飞书  /  终端  /  网页  /  Discord  / ...       │    │
│   └────────────────────────────────────────────────────┘    │
│                         │                                     │
│   ┌─────────────────────▼───────────────────────────────┐    │
│   │              Cron Scheduler                          │    │
│   │   定时任务 / 主动推送 / 周期性检查                   │    │
│   └────────────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────────────┘
```

**核心特点**：
- **框架级设计**：不是单一工具，是一个 Agent 开发和运行框架
- **记忆层**：长期记忆（MEMORY.md）+ 每日记忆（memory/YYYY-MM-DD.md）+ 知识库
- **Skill 系统**：每个技能是独立的 SKILL.md，可插拔、可共享
- **多渠道**：飞书、终端、网页、Discord 等，一个 Agent 多端触达
- **主动服务**：Cron 定时触发，主动给用户发消息，不需要用户来问
- **Provider 无关**：支持 MiniMax / OpenAI / Claude / Gemini 等多种模型
- **工具层**：Function Calling + 自定义工具（文件操作、代码执行等）

## 三、核心机制深度解析

### 3.1 工具系统（Tool System）

| | Claude Code | OpenCode | OpenClaw |
|---|---|---|---|
| **工具类型** | 内置 + MCP | 内置 + MCP | Function Calling + 自定义 Python 工具 |
| **工具粒度** | 粗粒度（Read/Write/Bash） | 细粒度（多种专业工具） | 可自定义（任何 Python 函数） |
| **扩展方式** | MCP 协议 | MCP 协议 | SKILL.md + Python 代码 |
| **工具上限** | 有限工具集 | 75+ LLM 提供商 | 取决于 Skill 实现 |

**Claude Code 的工具哲学**：窄而深。每个工具有明确的边界和副作用约束，工具越少越安全、越容易审计。

**OpenCode 的工具哲学**：宽而广。通过 MCP 连接任意工具，保持 core 小而精，扩展靠 MCP。

**OpenClaw 的工具哲学**：灵活自定义。Python 写什么工具就是什么工具，不受预设限制。

### 3.2 记忆系统（Memory System）

这是三者最核心的差异点。

**Claude Code**：
```
会话内存 = 上下文窗口（全部在 prompt 里）
         └── 关闭终端 → 全部丢失
```

**OpenCode**：
```
会话内存 = 单次连接内的上下文
项目记忆 = 可选（通过 /memory 命令手动保存）
跨会话 = ❌ 完全丢失
```

**OpenClaw**：
```
会话内存 = 上下文窗口
每日记忆 = memory/YYYY-MM-DD.md（每个工作日独立记录）
长期记忆 = MEMORY.md（跨时间、跨项目的核心记忆）
知识库   = 知识文件（markdown / json / 向量）
主动回忆 = 每次会话启动时读取 MEMORY.md
```

**举例**：
```
Day 1:
  用户：我开始做 CodeMind 项目了
  我：写入 memory/2026-03-22.md

Day 30:
  用户：CodeMind 现在什么进度？
  我：从 MEMORY.md 读取"CodeMind 项目在 ~/AIAgent/code-analysis-tool/"
  用户不需要重新描述项目背景
```

### 3.3 通信协议

| | Claude Code | OpenCode | OpenClaw |
|---|---|---|---|
| **主协议** | Anthropic Messages API | gRPC + WebSocket | WebSocket + REST |
| **工具协议** | Tool Use（Anthropic 专用格式） | MCP（开放标准） | Function Calling（OpenAI 兼容格式） |
| **扩展性** | 依赖 Anthropic 更新 | 完全开放 | 通过 Skills 扩展 |

**MCP（Model Context Protocol）**：
这是 OpenCode 和 OpenClaw（部分）共用的开放协议，允许 AI 连接到各种外部工具（GitHub、Slack、数据库等）。Anthropic 也在推动 Claude 对 MCP 的支持。

### 3.4 任务执行模式

**Claude Code**：
```
用户输入 → 实时响应 → 执行 → 反馈 → 继续（循环）
     ↑
  短反馈周期，AI 小步快跑
```

**OpenCode**：
```
用户输入 → AI 完整规划 → 后台执行 → 自动提交
     ↑
  长任务一次性执行，可以离开
```

**OpenClaw**：
```
用户输入 / 定时触发 → 规划 → 执行 → 记忆 → 推送结果
     ↑
  闭环：执行完还主动告诉你结果
```

## 四、使用场景对比

### 场景 1：凌晨 2 点，启动一个大重构

| | Claude Code | OpenCode | OpenClaw |
|---|---|---|---|
| **可行性** | ❌ 需要守在终端 | ✅ 后台整晚跑 | ❌ 需要先触发 |
| **操作** | — | `opencode --project ./repo` 然后描述任务 | — |
| **结果** | — | 醒来代码已经改好并 commit | — |

### 场景 2：接手一个陌生的 Java 微服务项目

| | Claude Code | OpenCode | OpenClaw |
|---|---|---|---|
| **能力** | 能读取代码，但依赖 LLM 猜测 | 能读取代码，但依赖 LLM 猜测 | 能用 CodeMind 做精确的 JavaParser AST 分析 |
| **确定性** | 概率性（可能幻觉） | 概率性（可能幻觉） | 确定性的（JavaParser 解析） |
| **调用链** | 模糊理解 | 模糊理解 | 精确到行号和方法的完整链路追踪 |

### 场景 3：每天早上收到 AI 新闻推送

| | Claude Code | OpenCode | OpenClaw |
|---|---|---|---|
| **能力** | ❌ 无法主动触达 | ❌ 无法主动触达 | ✅ Cron + 飞书推送 |
| **实现** | — | — | `cron: 0 8 * * 1-5` 触发搜索脚本 → 推送 |

### 场景 4：三个月后回头做之前的一个项目

| | Claude Code | OpenCode | OpenClaw |
|---|---|---|---|
| **记忆** | ❌ 完全不记得 | ❌ 需要手动描述背景 | ✅ 记得项目背景、你的目标、上次进度 |
| **操作** | 重新描述所有上下文 | 重新描述所有上下文 | 直接问："CodeMind 现在什么进度？" |

## 五、举例说明

### 例子：在项目中添加一个用户登录功能

#### Claude Code
```
$ claude-code
> 帮我加一个用户登录接口，需要 JWT token

# AI 立即开始：
1. 分析现有 User 模型
2. 写 LoginController.java
3. 写 AuthService.java  
4. 写单元测试
5. 问："要我直接提交吗？"
   → 你说"是"
   → 提交完成

# 关闭终端，下次打开，完全不记得这件事
```

#### OpenCode
```
$ opencode ./my-app

> 帮我加一个用户登录接口，需要 JWT token

# AI 分析后：
- 创建一个 todo list
- 后台依次实现每个部分
- 每小时自动 commit 一次
- 半夜 3 点完成，给你发个完成通知
- 你第二天早上来：代码已经写好

> /review 审查一下
- 自动分析 diff，列出潜在问题
```

#### OpenClaw（通过 CodeMind）
```
> 帮我追踪一下 /api/user/login 这个接口的完整调用链

# AI 用 CodeMind：
1. JavaParser 解析 Controller
2. 找到 @PostMapping("/api/user/login")
3. 追踪到 AuthService.login()
4. 追踪到 UserRepository
5. 输出完整的确定性的调用链报告

# 同时：
- 记住你问了这个问题
- 下次可以直接问"上次追踪的结果在哪"
```

## 六、架构设计哲学总结

| | Claude Code | OpenCode | OpenClaw |
|---|---|---|---|
| **哲学** | 做 AI 能做的最有用的事 | 让 AI 编程普惠开源 | 让 AI 成为真正的长期助手 |
| **核心假设** | 工具要少而精，安全可审计 | 工具要开放透明 | 记忆比工具更重要 |
| **时间维度** | 专注当下这一刻 | 专注当下 + 持久运行 | 跨时间、跨任务连续 |
| **用户角色** | AI 是工具，人是操控者 | AI 是工具，人是操控者 | AI 是助手，和人协作 |
| **扩展路径** | Anthropic 官方更新 | 社区 + MCP 生态 | Skills + 记忆 + 主动服务 |

---

## 七、总结

**本质上**：
- Claude Code = 最好的"当前"编程工具
- OpenCode = 最开源的"持久"编程工具
- OpenClaw = 最懂你的"长期"助手

**一句话评价**：
> Claude Code 是 AI 编程的"即时贴"，OpenCode 是"后台工人"，OpenClaw 是"记得一切的秘书"。

**互相取代？**
理论上，调教到极致，任何一个都可以模拟另一个。但架构设计决定了效率——让 Claude Code 记住你三年内的所有项目上下文，它需要极其长的 prompt，而 OpenClaw 从设计上是为此而生的。
